혜택on 관리자 사용 설명서

최종 업데이트: 2026.06.20  ·  대상: 사이트 운영자

1. 개요

혜택on 관리자 페이지(admin/index.html)는 사이트의 정책 데이터·기준값·자가진단 규칙을 코드 수정 없이 관리할 수 있는 도구입니다.

관리자 페이지에서 할 수 있는 것
기능설명
기준값 편집최저임금·기초연금·에너지바우처 등 매년 바뀌는 수치를 업데이트
정책 목록 관리정책 상태(운영중·만료·준비중)·마감일·출처 URL 수정
새 정책 추가새 정책의 HTML 파일·카드 코드를 자동으로 생성
자가진단 규칙연령·가구형태·고용상태에 따른 혜택 추천 로직 수정
데이터 검증오류·만료 정책·미업데이트 항목 자동 점검
내보내기/가져오기JSON 파일 다운로드 후 Git 반영
💡 저장 방식: 관리자에서 저장하면 브라우저의 localStorage에 먼저 저장됩니다. 사이트는 localStorage → JSON 파일 순서로 데이터를 읽기 때문에, 저장 즉시 사이트에 반영됩니다. Git에 영구 반영하려면 내보내기 → 파일 덮어쓰기 → push가 필요합니다.

2. 파일 구조

사이트의 주요 파일 구조입니다. 관리자는 아래 파일들을 직접 또는 관리자 페이지를 통해 관리합니다.

benefitson/
├── index.html                ← 메인 홈페이지
├── style.css                 ← 전체 스타일
├── main.js                   ← 메인 스크립트 (자가진단 등)
│
├── pages/                    ← 카테고리·아티클 페이지
│   ├── youth.html            ← 청년정책 카테고리
│   ├── newlywed.html         ← 신혼·육아 카테고리
│   ├── senior.html           ← 중장년·노년 카테고리
│   ├── article-XXX.html      ← 각 정책 아티클
│   └── calc-XXX.html         ← 계산기 페이지
│
├── data/                     ← 데이터 파일 (★ 관리자에서 편집)
│   ├── constants.json        ← 기준값 (최저임금·기초연금 등)
│   ├── policies.json         ← 정책 목록 (상태·마감일·출처)
│   ├── matching-rules.json   ← 자가진단 매칭 규칙
│   └── policy-loader.js      ← 데이터 로더 (수정 불필요)
│
└── admin/                    ← 관리자 페이지
    ├── index.html            ← 관리자 메인
    └── guide.html            ← 이 문서

3. 대시보드

관리자 페이지를 열면 첫 화면으로 표시됩니다. 사이트 데이터의 전반적인 상태를 한눈에 확인합니다.

카드의미조치
오류즉시 수정이 필요한 문제 (마감 지난 active 정책 등)검증 탭에서 상세 내용 확인 후 수정
경고90일 이상 미업데이트 정책 등 점검 권장 항목해당 정책 아티클 내용 검토 후 날짜 갱신
운영중 정책active 상태인 정책 수 / 전체 정책 수-
마감 미처리마감일이 지났지만 상태가 active인 정책 수정책 목록 탭에서 해당 정책을 expired로 변경

연간 업데이트 캘린더

정책별로 연간 업데이트 시점이 정해져 있습니다. D-일 표시가 빨간색이면 업데이트 시기가 임박한 것입니다.

정책업데이트 시점업데이트 내용
⚖️ 최저임금매년 8월 5일경다음 연도 시급 고시 → 기준값 탭에서 수정
👴 기초연금매년 1월인상률 적용 후 최대 지급액·선정기준액 변경
📊 기준중위소득매년 8월다음 연도 기준 고시
💵 근로장려금매년 4~5월신청 기간 시작 전 금액·소득 기준 확인
⚡ 에너지바우처매년 6~7월지원 금액·신청 기간 확인
📋 실업급여 상한액최저임금 연동최저임금 변경 시 자동 재계산됨

4. 기준값 편집

사이트 전체에서 사용하는 숫자 데이터를 관리합니다. 저장 즉시 사이트 새로고침 없이 반영됩니다.

1
기준값 탭 클릭
사이드바에서 ⚙️ 기준값을 클릭합니다.
2
수치 수정
변경된 수치를 입력합니다. 월 환산액·인상률·실업급여 하한액은 입력값을 바탕으로 자동 계산됩니다.
3
💾 저장 (즉시 반영) 클릭
localStorage에 저장되며 사이트에 바로 반영됩니다. 새로고침 불필요.
4
Git에 영구 반영
내보내기 탭에서 constants.json을 다운로드 → data/ 폴더에 덮어쓰기 → Git push
⚠️ 주의: 최저시급은 실업급여 하한액과 연동됩니다. 최저시급 변경 후 실업급여 상한액이 하한액보다 낮아지면 오류가 발생하므로, 함께 확인해주세요.

5. 정책 목록 관리

등록된 모든 정책의 상태·마감일·출처를 관리합니다. 마감일이 지난 정책은 빨간색으로 표시됩니다.

정책 상태 종류

상태의미언제 사용
active현재 신청 가능하거나 상시 운영 중정책이 유효한 동안
expired마감 또는 폐지된 정책신청 기간 종료 후
draft준비 중, 사이트에 아직 미노출아티클 작성 전 임시 등록 시

정책 편집 방법

1
정책 목록 탭 → 해당 정책 행의 편집 클릭
2
내용 수정 후 저장 클릭
마감일·상태·출처 URL·요약 등을 수정합니다.
3
하단 💾 정책 저장 클릭
편집 내용이 localStorage에 최종 저장됩니다. 이 버튼을 누르지 않으면 페이지를 닫을 때 변경 사항이 사라집니다.
✅ 빠른 만료 처리: 편집 패널의 만료 처리 버튼을 클릭하면 상태가 즉시 expired로 변경됩니다.

6. 새 정책 추가 ★

새로운 국가정책을 사이트에 추가하는 전체 흐름입니다. 관리자 페이지가 3가지를 자동으로 생성해줍니다.

💡 새 정책 하나를 추가할 때 필요한 작업:
① policies.json에 항목 등록  →  ② 아티클 HTML 파일 작성  →  ③ 카테고리 페이지에 카드 추가  →  ④ Git push

Step 1 — 기본 정보 입력

사이드바 ➕ 새 정책 추가를 클릭하고 아래 항목을 입력합니다.

항목입력 예시설명
정책 제목 필수2026 청년도약계좌 가입 조건아티클 페이지의 H1 제목이 됩니다
파일명 필수article-youth-account카테고리 변경 시 자동 제안됨. pages/ 폴더에 저장될 이름 (.html 제외)
카테고리 필수청년정책청년정책·신혼·육아·중장년·노년·1인가구·근로/소득·세금/환급 중 선택
상태activeactive(운영중) 또는 draft(준비중)
마감일2026-08-31상시 운영이면 비워두기
출처 URLhttps://www.bokjiro.go.kr정부 공식 신청 사이트 주소
한줄 요약 필수연 최대 5,000만원, 정부기여금 144만원아티클 상단 요약 박스에 표시됩니다
태그청년, 적금, 자산형성쉼표로 구분. 검색·필터에 활용

Step 2 — ① policies.json에 등록

1
목록에 추가 버튼 클릭
정책 목록 탭의 테이블에 새 항목이 추가됩니다.
2
정책 목록 탭 → 💾 정책 저장 클릭
localStorage에 최종 저장됩니다. 이 버튼을 눌러야 데이터가 유지됩니다.

Step 3 — ② 아티클 HTML 파일 다운로드

1
HTML 다운로드 버튼 클릭
입력한 제목·요약·카테고리가 채워진 HTML 파일이 자동 생성되어 다운로드됩니다.
2
다운로드 파일을 pages/ 폴더에 저장
예: pages/article-youth-account.html
3
파일을 열어 내용 채우기
<!-- 신청 자격 및 대상 조건을 작성하세요 --> 형태의 주석을 찾아 실제 내용으로 교체합니다.
주요 작성 구간: 지원 대상 / 지원 내용 / 신청 방법 / 자주 묻는 질문

Step 4 — ③ 카테고리 카드 코드 추가

1
코드 보기 버튼 클릭 → 📋 복사
카테고리 페이지용 카드 코드와 index.html용 카드 코드 두 가지가 생성됩니다.
2
카테고리 페이지에 붙여넣기
예: 청년정책이면 pages/youth.html을 열고 .articles-main-grid 태그 안에 복사한 코드를 붙여넣습니다.
3
index.html 메인 섹션에도 추가 (선택)
주목도 높은 정책은 index.html의 해당 카테고리 섹션 .articles-grid 안에도 카드 코드를 추가합니다.

Step 5 — Git push로 배포

아래 10. 배포 방법 섹션을 참고해 변경 파일을 push합니다.

7. 자가진단 규칙

메인 홈페이지의 자가진단 기능에서 연령·가구형태·고용상태 조합에 따라 어떤 혜택을 추천할지 정의합니다.

규칙 JSON 구조
{
  "benefitsMap": {
    "20대_1인가구_직장인": [
      { "title": "청년월세 특별지원",  "url": "pages/article-youth-rent.html" },
      { "title": "실수령액 계산기",     "url": "pages/calc-net-salary.html" }
    ],
    "30대_자녀 있는 가구_직장인": [
      { "title": "육아휴직 급여",       "url": "pages/article-parental-leave.html" },
      { "title": "아이돌봄 바우처",     "url": "pages/article-childcare.html" }
    ]
  }
}

키(Key) 형식

키는 연령대_가구형태_고용상태 형식으로 언더스코어(_)로 연결합니다.

항목가능한 값
연령대10대, 20대, 30대, 40대, 50대, 60대 이상
가구형태1인가구, 부모님과 동거, 배우자와 동거, 자녀 있는 가구
고용상태직장인, 취업준비중, 자영업자, 무직·기타
⚠️ JSON 문법 오류(쉼표 누락, 따옴표 불일치 등)가 있으면 저장이 되지 않습니다. 문법 검사 버튼으로 미리 확인하세요.

사이드바 ✅ 검증 탭에서 자동으로 아래 항목들을 점검합니다.

점검 항목오류 조건해결 방법

9. 내보내기 / 가져오기

관리자에서 편집한 데이터를 data/ 폴더의 JSON 파일로 영구 반영하는 방법입니다.

내보내기 (localStorage → 파일)

1
내보내기 탭에서 원하는 파일 다운로드
⬇ constants.json ⬇ policies.json ⬇ matching-rules.json
2
다운로드 파일을 프로젝트 data/ 폴더에 덮어쓰기
3
Git push
아래 배포 방법 섹션 참고

가져오기 (파일 → localStorage)

다른 기기에서 편집한 JSON 파일을 업로드하거나, 백업 파일을 복원할 때 사용합니다. 📂 파일 선택 후 JSON 파일을 선택하면 localStorage가 해당 내용으로 교체됩니다.

⚠️ localStorage 초기화: 초기화하면 파일에서 데이터를 다시 불러옵니다. 저장하지 않은 편집 내용이 모두 사라집니다.

10. 배포 방법

새 정책 추가·아티클 수정·JSON 파일 업데이트 후 사이트에 반영하는 방법입니다.

변경 파일 유형별 처리

변경 내용필요 작업
기준값(최저임금 등) 수정관리자 저장(즉시 반영) + constants.json 내보내기 후 Git push
정책 상태·마감일 수정관리자 저장(즉시 반영) + policies.json 내보내기 후 Git push
새 아티클 HTML 추가파일을 pages/에 저장 + 카테고리 페이지 카드 추가 + Git push
아티클 내용 수정해당 HTML 파일 직접 수정 + Git push

Git push 방법 (Claude Code 터미널)

# 1. 변경된 파일 확인
git status

# 2. 변경 파일 스테이징 (예시)
git add data/policies.json pages/article-XXX.html pages/youth.html

# 3. 커밋
git commit -m "feat: XXX 정책 추가"

# 4. push (GitHub Pages 자동 배포)
git push origin main
✅ push 후 약 1~2분 내에 GitHub Pages에 자동 배포됩니다. GitHub 저장소의 Actions 탭에서 배포 진행 상태를 확인할 수 있습니다.

11. 자주 묻는 질문

Q. 관리자에서 저장했는데 사이트에 반영이 안 됩니다.

같은 브라우저에서 사이트를 열었는지 확인하세요. localStorage는 브라우저·도메인 단위로 저장되므로, 다른 브라우저나 다른 컴퓨터에서는 변경사항이 보이지 않습니다. 다른 기기에 반영하려면 내보내기 → Git push가 필요합니다.

Q. 새 카테고리(네비게이션 메뉴)를 추가하려면?

현재 네비게이션 추가는 HTML 파일을 직접 수정해야 합니다. 모든 HTML 파일의 <nav class="nav-main">과 <nav class="mobile-menu">에 항목을 추가하고, 해당 카테고리 페이지(예: pages/newcategory.html)를 생성해야 합니다. Claude Code에 요청하면 일괄 처리해 드립니다.

Q. 아티클 HTML 파일의 내용은 어떻게 작성하나요?

다운로드된 HTML 파일을 텍스트 편집기(VS Code 등)로 열면 <!-- 신청 자격 및 대상 조건을 작성하세요 -->와 같은 주석이 있습니다. 이 주석을 실제 내용으로 교체합니다. 기존 아티클(예: article-parental-leave.html)을 참고하면 구조를 파악하기 쉽습니다.

Q. policies.json의 "id"는 무엇인가요?

정책을 고유하게 식별하는 영문 식별자입니다. 파일명에서 article-를 뺀 값을 사용합니다. 예: 파일명이 article-youth-rent.html이면 id는 youth-rent.

Q. localStorage 데이터가 사라졌습니다.

브라우저 캐시 삭제, 시크릿 모드 사용, localStorage 용량 초과 등의 이유로 데이터가 사라질 수 있습니다. 이것이 이유로 정기적으로 내보내기 → Git push로 JSON 파일에 영구 반영해두는 것을 권장합니다.
